## What's new - v.4.8.1

- Interaction modes (plan/auto) — REPL sessions now start in plan mode by default (one-shot runs stay in auto). Plan mode blocks file edits, bash, and sub-agent spawning while allowing all read-only tools. Press Shift+Tab at the prompt to toggle modes; the choice persists across restarts via config.                                       
- Tool-argument repair — New parseToolArguments / repairToolArguments handles malformed LLM
output (code fences, trailing commas, truncated braces) instead of hard-failing on bad JSON.
- Result-ref IDs in streaming output — tool_end chunks now carry resultRefId so the REPL can
display [result #N] inline, and the raw (untruncated) content is what gets cached to disk. 
- web(fetch) reclassified as read-only — No longer requires a permission prompt; both
web(fetch) and web(search) are auto-allowed.                                                
- /clear alias & session-level clear — /clear (defaults to session) resets conversation state while keeping memory, plans, tasks, and history. /memory clear session does the same. 
Completion engine updated with scope suggestions in safer order.  
- System prompt overhaul — Expanded with retrieval-policy ordering, task discipline, exactness rules, and verification guidance.                                                 
- Grep tool: execFile → execFileSync — Switched ripgrep invocation to synchronous;pattern
now passed after flags (fixing a positional arg issue); empty results return "" instead of  
"No matches found.".                                              
- Footer shows current mode — Mode label (plan/auto) rendered in green or orange before the model name.
