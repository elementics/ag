## What's new - v4.8.3

- Self tool — New built-in `self` tool for managing custom tools, skills, and extensions. Supports `list`, `create`, `edit`, `remove`, `disable`, and `enable` actions. On first run, bootstraps `~/.ag/ag.md` from the bundled asset as a scaffolding template users can edit freely.
- Bash cancellation — Foreground bash processes now respect abort signals. Sends SIGTERM on interrupt, SIGKILL after a 3-second grace period, and returns EXIT 130. The agent execution loop adds an abort race so any long-running tool call can be cancelled cleanly.
- Context token count in footer — Status bar now shows the current prompt token count before the context bar, color-coded green (< 30K), yellow (≥ 30K), orange (≥ 50K), and red (≥ 90K).
- Sub-agent result refs — Result refs created during sub-agent execution are now surfaced to the parent agent via a ref appendix. New `returnRaw` parameter on the `agent` tool skips summarization and returns the sub-agent's last response verbatim — useful for content-fetching tasks.
- Scope-aware skill management — `/tools` and `/skill` listings now show a `[project]` tag for project-local items. `removeSkill` checks project scope before global and reports which was removed. Registry lookup errors now suggest a manual install path instead of a bare not-found message.
- `/clear session` context re-estimate — Token count is re-estimated from current context after a session clear, fixing stale context bar readings.
- Checkpoint session ID always shown — Current session ID is now rendered in checkpoint listings, not only for cross-session checkpoints.
- Removed `bash via skill` label — Tool completion display no longer appends `via SkillName` for bash calls made during an active skill. The label was unreliable (skills never deactivate, attribution used last-skill-wins, only bash was affected). Removed dead `getActiveSkillNames()`.
