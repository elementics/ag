## What's new - v4.8.2

- Loop ledger and repeat-edit guard — ag now tracks files changed, reads, verification commands,and guard decisions during each turn. If the agent tries to edit/write the same file more than three times without reading it or running verification, the turn stops instead of continuing a blind edit loop.
- Stronger edit/write evidence — file(write) and file(edit) now return bounded current-content snippets after mutation. file(edit) also rejects exact no-op edits, making failed recovery loops easier to diagnose.
- Checkpoint UX improvements — /checkpoint now lists newest first. /rewind shows a diff stat preview of code changes since the selected checkpoint before asking whether to restore code, conversation, or both.
- Shadow rename — Internal checkpoint snapshot code moved from shadow-git.ts to shadow.ts, and the project cache folder is now shadow/ instead of shadow-git/. No migration path is included.
- JSONL traces — ag now writes per-session JSONL traces under the project cache. Trace files are pruned automatically: latest 20 files kept, files older than 14 days removed, active traces rotated after 10MB.
- Better compaction fidelity — Compaction now preserves useful tool-call arguments such as file paths, bash commands, grep queries, and git actions instead of only recording tool names.
- Better result summaries — File mutation result refs now include a short evidence preview, and bash summaries correctly detect EXIT N at the start of failed command output.
- Resume/context fixes — Session resume now preserves recent working-state context, and async turn summaries are attributed to the correct turn even if later turns start before summarization completes.
